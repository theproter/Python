Funciones en Python

Una función es un bloque de código reutilizable que realiza una tarea específica. Se define utilizando la palabra clave def. Aquí hay un ejemplo básico:

def suma(a, b):
    return a + b

resultado = suma(3, 5)
print(resultado)  # Output: 8

Parámetros y Argumentos

Los parámetros son variables que se utilizan para pasar información a una función, mientras que los argumentos son los valores reales que se pasan a la función al llamarla. Python permite definir funciones con parámetros posicionales, parámetros con valores predeterminados, y parámetros de longitud variable. Aquí hay un ejemplo:

def saludar(nombre, mensaje="Hola"):
    print(f"{mensaje}, {nombre}!")

saludar("Juan")  # Output: Hola, Juan!
saludar("María", "Buenos días")  # Output: Buenos días, María!

Funciones Anónimas (Lambda)

En Python, las funciones anónimas (lambda) son funciones pequeñas y anónimas que pueden tener cualquier número de argumentos pero solo pueden tener una expresión. Se definen utilizando la palabra clave lambda. Aquí hay un ejemplo:

doble = lambda x: x * 2
print(doble(5))  # Output: 10

Funciones Recursivas

Una función recursiva es una función que se llama a sí misma para resolver un problema. Es importante tener en cuenta el caso base para evitar que la función entre en un bucle infinito. Aquí hay un ejemplo de una función recursiva para calcular el factorial de un número:

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n - 1)

print(factorial(5))  # Output: 120

Funciones de Orden Superior

En Python, las funciones son ciudadanos de primera clase, lo que significa que pueden ser tratadas como cualquier otra variable. Esto permite pasar funciones como argumentos a otras funciones, y devolver funciones desde otras funciones. Aquí hay un ejemplo:

def aplicar_funcion(func, lista):
    return [func(x) for x in lista]

cuadrado = lambda x: x ** 2
numeros = [1, 2, 3, 4, 5]
print(aplicar_funcion(cuadrado, numeros))  # Output: [1, 4, 9, 16, 25]

Programación Orientada a Objetos en Python

La programación orientada a objetos (POO) es un paradigma de programación que utiliza "objetos" para modelar datos y comportamientos. En Python, todo es un objeto, lo que significa que cada entidad en el programa tiene sus propios atributos (datos) y métodos (funciones).

Clases y Objetos

Una clase es una plantilla para crear objetos, y un objeto es una instancia de una clase. Las clases definen las propiedades (atributos) y los comportamientos (métodos) de los objetos. Aquí hay un ejemplo básico:

class Persona:
def __init__(self, nombre, edad):
self.nombre = nombre
self.edad = edad

def saludar(self):
print(f"Hola, mi nombre es {self.nombre} y tengo {self.edad} años.")

# Crear un objeto de la clase Persona
persona1 = Persona("Juan", 25)
persona1.saludar()  # Output: Hola, mi nombre es Juan y tengo 25 años.

Herencia

La herencia es un concepto clave en la POO que permite a una clase (llamada clase derivada o subclase) heredar atributos y métodos de otra clase (llamada clase base o superclase). Esto permite la reutilización de código y la creación de jerarquías de clases. Aquí hay un ejemplo:

class Empleado(Persona):
def __init__(self, nombre, edad, salario):
super().__init__(nombre, edad)
self.salario = salario

def mostrar_salario(self):
print(f"Mi salario es {self.salario}")

# Crear un objeto de la clase Empleado
empleado1 = Empleado("Ana", 30, 50000)
empleado1.saludar()  # Output: Hola, mi nombre es Ana y tengo 30 años.
empleado1.mostrar_salario()  # Output: Mi salario es 50000

Encapsulación, Polimorfismo y Abstracción

Además de la herencia, la POO se basa en otros tres conceptos principales:

  • Encapsulación: El ocultamiento de los detalles de implementación de un objeto dentro de su clase.
  • Polimorfismo: La capacidad de los objetos de una misma clase para responder de manera diferente a los mismos mensajes.
  • Abstracción: La simplificación de un problema modelando solo los aspectos importantes y ignorando los detalles innecesarios.

Métodos Especiales

En Python, los métodos especiales (también conocidos como métodos mágicos o dunder methods) permiten definir comportamientos específicos para objetos, como la inicialización (__init__()), la representación (__str__()), la suma (__add__()), entre otros.

class Punto:
def __init__(self, x, y):
self.x = x
self.y = y

def __str__(self):
return f"({self.x}, {self.y})"

def __add__(self, other):
return Punto(self.x + other.x, self.y + other.y)

# Crear objetos de la clase Punto
punto1 = Punto(1, 2)
punto2 = Punto(3, 4)
print(punto1 + punto2)  # Output: (4, 6)
